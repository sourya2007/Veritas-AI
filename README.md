# Veritas AI
https://sourya2007.github.io/Veritas-AI/
Veritas AI is a full-stack fake-news analysis platform with:

- Live global news feed ingestion from RSS sources
- Claim verification endpoint with evidence aggregation
- Local TensorFlow model inference for fake-vs-reliable classification
- React dashboard for feed browsing, verification, and model showcase

## Tech Stack

- Frontend: React 19, TypeScript, Vite, React Router
- Backend: FastAPI, Pydantic
- ML: TensorFlow/Keras, scikit-learn, pandas, numpy
- Data: `True.csv` and `Fake.csv` (tracked with Git LFS)

## Repository Layout

- [frontend](frontend): Vite + React web app
- [backend](backend): FastAPI app and API routes
- [ml/artifacts](ml/artifacts): metrics and model artifacts generated by training
- [final_train.py](final_train.py): trains local LSTM model on `True.csv` + `Fake.csv`
- [simple_predict.py](simple_predict.py): runs local inference against trained artifacts
- [showcase_local_model.py](showcase_local_model.py): CLI demonstration against running backend APIs
- [detector_model.keras](detector_model.keras): primary trained model artifact
- [detector_model.h5](detector_model.h5): optional secondary model artifact
- [detector_tokenizer.json](detector_tokenizer.json): tokenizer artifact
- [detector_config.json](detector_config.json): runtime config (max_len, threshold, selected model file)

## Prerequisites

- Node.js 20+ and npm
- Python 3.11 recommended for TensorFlow support
- Git LFS installed

### Important Python Note

TensorFlow wheels are not available for Python 3.14 in this setup. Use Python 3.11 for model training and local-model inference mode.

## Backend Setup

From repository root:

```powershell

```bash
python3.11 -m venv .venv-tf
source .venv-tf/bin/activate
pip install -r backend/requirements.txt
cd backend
python -m uvicorn app.main:app --reload --port 8000
```

Health check:

```bash
curl http://localhost:8000/health
```

## Frontend Setup

In a new terminal from repo root:

```bash
cd frontend
npm install
npm run dev
```

Frontend runs on Vite default (usually `http://localhost:5173`).

### API Base URL Override

Frontend uses `VITE_API_BASE_URL` and defaults to `http://localhost:8000`.

Create `frontend/.env.local` if needed:

```env
VITE_API_BASE_URL=http://localhost:8000
```

If backend is running on another port (for example 8012), update that value.

## Running the Full App

1. Start backend first (`uvicorn` on desired port).
2. Start frontend (`npm run dev`).
3. Open the app in browser.
4. Use:
    - Home feed for live articles
    - AI Verification page for claim verification
    - Model Showcase page for local model metrics and demo inference

## Deployment (Vercel)

### Recommended architecture

- Deploy `frontend` to Vercel
- Deploy `backend` to a Python host (Render/Railway/Fly/VM)
- Set `VITE_API_BASE_URL` in Vercel to your backend public URL

This project uses TensorFlow model artifacts, which are typically not suitable for Vercel Serverless limits. Frontend-on-Vercel + backend-on-Python-host is the reliable production setup.

### Deploy frontend to Vercel

1. Push repository to GitHub (already done).
2. In Vercel, click **Add New Project** and import this repository.
3. Set **Root Directory** to `frontend`.
4. Build settings:
   - Framework preset: `Vite`
   - Build command: `npm run build`
   - Output directory: `dist`
5. Add environment variable in Vercel Project Settings:
   - `VITE_API_BASE_URL=https://<your-backend-domain>`
6. Deploy.

### Important notes

- `frontend/vercel.json` is included for SPA routing, so routes like `/verify` and `/model-showcase` resolve correctly.
- API client now defaults to same-origin only in production if `VITE_API_BASE_URL` is not set, and to `http://localhost:8000` in local development.

## Backend Runtime Modes

Model service runs in one of two modes:

- `local-model`: TensorFlow model + tokenizer loaded successfully
- `heuristic-fallback`: fallback lexical heuristic if model dependencies/artifacts unavailable

Check mode via:

```bash
curl http://localhost:8000/api/model/metrics
```

## API Reference

### Health

- `GET /health`

Response:

```json
{
   "status": "ok",
   "service": "veritas-backend"
}
```

### Feed

- `GET /api/feed`
- Optional query params: `topic`, `genre`, `source`, `search`, `page`, `page_size`

Example:

```bash
curl "http://localhost:8000/api/feed?search=election&page=1&page_size=12"
```

### Article by id

- `GET /api/articles/{article_id}`

### Verify claim

- `POST /api/verify`

Request body:

```json
{
   "claim_text": "Government confirms national inflation dropped this quarter"
}
```

### Model inference

- `POST /api/model/infer`

Request body:

```json
{
   "text": "HOAX: Government to seize all private savings accounts starting Monday morning."
}
```

### Model metrics

- `GET /api/model/metrics`

Includes:

- `model_name`
- `dataset`
- `metrics` (`f1`, `precision`, `recall`, `roc_auc`)
- `overall_score`
- `mode`
- `prediction_count`
- `last_prediction_ms`
- `avg_prediction_ms`

## Troubleshooting

### Backend fails to start with TensorFlow errors

- Ensure Python 3.11 environment is active
- Reinstall backend deps in that environment
- On Windows, install Microsoft Visual C++ Redistributable if DLL errors occur

### Port conflicts (8000 already in use)

- Start backend on another port:

```bash
python -m uvicorn app.main:app --reload --port 8012
```

- Then set frontend API URL accordingly in `frontend/.env.local`.

### Frontend can’t reach backend

- Confirm backend is running and reachable at configured `VITE_API_BASE_URL`
- Check CORS/network errors in browser dev tools

### Model metrics show heuristic fallback

- Confirm artifacts exist in repo root
- Confirm TensorFlow import works in the active backend environment
- Confirm `detector_config.json` points to an existing model file
